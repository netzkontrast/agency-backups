# Governance Specs Update Research Prompt

**What and Why:**
This folder contains the prompt that directs an agent to research the current state of root specs and tooling, to formulate a detailed update plan. This is necessary because Task 001 introduced new tools and mechanics, which may not be fully documented or may conflict with the old specs.

**Navigation:**
- [`brief.md`](./brief.md): The raw user request.
- [`prompt.md`](./prompt.md): The executable research proposal.

**Assumptions:**
We assume that the tooling implemented in Task 001 (e.g. `check-governance.sh`) is the desired target state and the specs need to catch up.

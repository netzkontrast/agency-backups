# Cross-Pollination: Category A → Category C

> Inject into a **Lifecycle** prompt (primary = C). Guards against the
> core C failure mode: the persistent knowledge store being extended
> month after month without ever re-opening the question of whether
> the underlying framing is still the right framing.

## Paste-Ready Injection — One Step (minimum) into the Steps section

```markdown
### Step [i.a] — Framing Re-Opening (cross-pollination from Category A)

This step imports one exploratory challenge from Category A
(Exploration) because lifecycle research drifts slowly: the framing
inherited from session 1 tends to survive into session 20 unexamined,
and the accumulated notes lend it false legitimacy.

Perform the following at every [Nth] session (recommended: every
fourth, and any time the Assumption-Decay Audit fires):

1. **Competing-frame hypothesis.** Write down at least one credible
   **alternative framing** for the entire research question. Not a
   new hypothesis within the current frame — a different frame. If
   the current frame is "[FRAME X]", generate an alternative of the
   form "[FRAME Y that reinterprets the same phenomenon differently]".

2. **One orthogonal session.** For this session only, conduct the
   research as if the alternative frame were the active one. Record
   findings in a side-branch of the knowledge store (clearly marked
   as **Orthogonal Frame Trial [FRAME Y]**).

3. **Compare.** At the end of this session, compare: does the
   alternative frame surface any evidence the primary frame was
   blind to? If yes, escalate — the main framing may need revision.
   If no, log the trial as negative and continue in the primary frame.

4. **Do not hybridize.** Do not silently blend the two frames into
   the main knowledge store. The side-branch stays a side-branch
   until an explicit re-framing decision is made.
```

## When to Apply

At least every fourth session, plus any time the Assumption-Decay Audit
fires. Cost: one full session per application. Benefit: early detection
of dead-frame research, which is the dominant silent failure of long-
running monitoring projects.

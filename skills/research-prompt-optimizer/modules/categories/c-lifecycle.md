# Category C — Lifecycle · Paste-Ready Inline Block

> Source of truth for the **Epistemological Layer** block when Phase 2
> routes to Category C.

```markdown
## Epistemological Layer — Category C (Lifecycle)

This research is **ongoing**, not one-shot. It will be resumed across
multiple sessions, will accumulate material over weeks or months, and
must remain coherent across that timespan.

**What this means for your execution:**

1. **Maintain a persistent knowledge store.**
   You keep a structured notes file (format specified in the
   Expectations section) that survives session boundaries. At session
   end you save it; at session start you load it and re-anchor.

2. **Execute the Resumption Protocol at every session start.**
   Before doing any new research, you:
   (a) load the persistent knowledge store,
   (b) re-read the active research question verbatim,
   (c) restate the Constraint Blocks verbatim,
   (d) run the Assumption-Decay Audit (defined in methods) to verify
   that your foundational assumptions still hold,
   (e) only then proceed to new material.

3. **Compact older material regularly.**
   When the knowledge store exceeds [N tokens / N entries], compact the
   older half by summarizing it into a compact digest. Preserve all
   contradictions, sources, and flagged items verbatim; compact only
   descriptive narrative. Never silently delete.

4. **Surface contradictions that emerge across time.**
   In lifecycle research, contradictions often appear because
   **the world changed** between session 3 and session 7. Distinguish:
   (a) sources disagree about the same state of the world, vs.
   (b) the state of the world moved. Category (b) goes into a
   **World-Change Log**, not the regular Contradiction Log.

5. **Flag assumption decay loudly.**
   If the Assumption-Decay Audit finds that a foundational assumption
   no longer holds, halt the normal flow. Mark it as **AUDIT FAILURE**
   in the knowledge store. Re-evaluate which previous conclusions still
   stand. Do not continue extending the research on a broken foundation.

**Operational constraint:** Long-run stability beats short-run
throughput. A session that produces no new findings but successfully
updates assumptions is a successful session.
```

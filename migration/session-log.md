# Session log — the design conversation behind `/migration/`

This is the narrative reconstruction of what happened across multiple sessions to produce the 11 ratified locks now in [`locks-ratified.md`](./locks-ratified.md). The purpose is **handover continuity** — if the next session needs to understand "why this lock instead of that one", the answer lives here.

---

## Roundtable 7 (sessions prior to 2026-05-13)

**State entering:** 7 first-class types from L11.32′; 5 candidates (`role`, `lock`, `gherkin`, `friction-log`, `hook`) in limbo; Gemini brief #1 commissioned in Round 9 to audit 14 refactor decisions; Gemini brief #2 commissioned in Round 10 for architectural cross-validation.

**What was ratified:**

- **L11.32‴** — 12-type ontology adopted. The 5 candidates promoted to first-class. Driver: D1.
- **L11.36′** — three placement modes (STANDALONE / SUBFILE / SUBDOC) adopted. Driver: D2.
- **L11.37′** — SQLite uniform graph layer adopted. Drivers: D5 + D6.
- **L11.38′** — Pandoc fenced divs as SUBDOC syntax. Driver: D4.
- **L11.39′** — mode-aware CLI surface declared. Driver: D7.
- **L11.40′** — locks STANDALONE-only with `lock_sha` content-addressing. Driver: D3.
- **L11.41′** — hooks STANDALONE-only under `tools/hooks/<event>/<slug>/`.
- **L11.42** — closed schemas everywhere. Driver: D8.
- **L11.43 (original)** — ULID-prefixed folder convention for all 12 types except `adr`, shape `<slug>-<ulid>/`, archive-first big-bang migration, display `<slug>-<first-8-of-ulid>`.

**Output:** `.claude/plans/roundtable-7-ontology-locks-recap.md` captured all 9 locks plus Q2–Q6 as open issues blocking ADR drafting.

## Roundtable 8 — session of 2026-05-13

**State entering:** the recap from Roundtable 7. Q2–Q6 open. User opened the session asking for next steps; `AskUserQuestion` was the agreed-on interaction primitive for design rounds.

### Round 8a — L11.43 scope shrunk to tasks-only

The user re-opened L11.43 itself. Their original answer to "which types adopt ULID?" had been "all 12 except ADR". After reflection, they changed to **"only tasks initially"** — with the implicit understanding that other types don't carry the duplicate-NNN pain that motivated the change.

Cascade: the folder shape also revised from `<slug>-<ulid>/` to bare `<slug>/` with ULID in frontmatter only. This produced a Q2/Q4 tension (the "short-prefix + slug" display answer only makes sense when the ULID is somewhere visible). Three clarifying questions resolved it:

- **Display:** slug alone is canonical; auto-disambiguation appends short ULID only on collision.
- **Long-term scope:** permanently tasks-only — other types stay in their current schemes forever.
- **Archive names:** preserve original `<NNN>-<slug>` names; do not rename or retroactively mint ULIDs.

L11.43 was revised in place — same lock number, new content.

### Round 8b — Decision 4 (per-type natural-fit for promoted types)

With L11.43 narrowed, the question became "what identifies role / lock / gherkin / friction-log / hook?". The user picked **per-type natural fit** — each type keeps whatever convention it already de-facto uses.

Then two follow-up rounds nailed:

- **`role` home:** `roles/<slug>/role.md` — new top-level operational folder.
- **`lock` notation:** preserve user-facing `L<round>.<sub>` style (e.g. `L11.43`).
- `gherkin` / `friction-log` / `hook` deferred to Q6 detail.

### Round 8c — L11.44 introduced (and revised same-session)

The user surfaced a new requirement: folder readability + easy navigation between related artifacts. They proposed an `agency` CLI that generates a readme in each task folder mapping all related artifacts.

The agent proposed v1: auto-managed nav block inside marker comments, hand-written body for "Why this folder exists" prose + assumption log.

Four-question round nailed v1:

- **Scope:** all operational folders (not just tasks).
- **Trigger:** pre-commit auto-regenerate.
- **Edge direction:** bidirectional.
- **Empty handling:** render `(none)` rows.

Then immediately, the user **revised** to v2 in the next message: *"the readme.md files should all be autogenerated - everywhere - and the frontmatter of all artifacts Need to hold all the Information needed - that should e reflection in the Schemas"*.

L11.44 v2 = full machine-rendering; no hand-written body anywhere; schemas expand to carry the narrative content (`purpose`, `assumptions`).

### Round 8d — Pivot to `/migration/` folder + handover

The user's final instruction of the session: write everything into a new `/migration/` folder; create a handover doc; add banners to CLAUDE.md and AGENTS.md. The realisation: 11 locks plus Q1–Q7 plus schema deltas is too much surface to ratify in one session.

This file documents that decision and inventories the artifacts produced.

## Why the recap document was superseded

`.claude/plans/roundtable-7-ontology-locks-recap.md` was the plan-of-record after Roundtable 7. It captured L11.43 in its **original** form (all 12 types except ADR, `<slug>-<ulid>/` folder shape, etc.). Roundtable 8 revised L11.43 in two ways: scope shrunk to tasks-only, and the ULID moved from path to frontmatter. The recap document does not reflect these revisions.

Rather than mutate the recap (which would lose the historical record of what Roundtable 7 ratified), the recap is left in place as a historical snapshot. `/migration/` is the new plan-of-record; the recap is its ancestor.

A pointer at the top of the recap (added in this session) directs readers here.

## What did NOT happen this session
